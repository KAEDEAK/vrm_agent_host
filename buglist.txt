   - HTTP terminateと同じ終了パスを使用することで設定保存を確実に実行

   - タスクバーの「ウィンドウを閉じる」でWM_CLOSEメッセージを受信

根本原因の特定:
ユーザーの指摘「ウィンドウの終了に反応しなくなった時があったじゃん。あのタイミングで wing menu system と同じように やったんじゃだめなの？」により真の解決策を発見:

- 「ウィンドウの終了に反応しなくなった時」= Windows Message Handlerが機能していた時
- その時はWM_CLOSEでInvokeShutdown()を呼び出していた
- Wing Menu EXITと同じ処理パスを使用することが正解

技術的解決策:
Windows Message HandlerのWM_CLOSEでInvokeShutdown()を呼び出し:
```csharp
case WM_CLOSE:
    Debug.Log("🚪 [DEBUG] WM_CLOSE received - calling InvokeShutdown() like Wing Menu EXIT");
    
    // Wing Menu EXITと同じようにInvokeShutdown()を呼び出す
    InvokeShutdown();
    Debug.Log("🚪 [DEBUG] InvokeShutdown() called, returning IntPtr.Zero");
    return IntPtr.Zero; // Prevent default handling temporarily
```

実装の特徴:
- Windows Message Handlerを復活
- WM_CLOSEでInvokeShutdown()を呼び出し
- Wing Menu EXITと完全に同じ処理パス
- 全ての終了方法でApplication.Quit()が確実に実行される
