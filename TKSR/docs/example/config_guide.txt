-----------------------
config ファイル
-----------------------

config の 例です。

config は、以下のファイル名で複数配置できます。
実行ファイル VRMAgentHost-X1.exe と同じ場所においてください。

実行時に内部でマージされます。

後から読み込まれるものほど優先度が高くなります。

以下の順で読み込まれます。

- 内蔵のデフォルトの config
- config.json
- lastWindowBounds.json
- override_*.json
- override.json


おおもとの configを変更せず
適宜変更したいパラメータだけを記載したファイルを用意することで
迅速に config の切り替えができます。

-----------------------
config.json 
-----------------------
初回起動時に出力されます。
ほとんどの設定がデフォルト値で網羅的に記載されています。

-----------------------
override.json
-----------------------
おすすめは override.json を用意して運用する方法です。
config.json は参照専用とすると便利です。
以下のように複数利用できます。
順にマージされます。最後に override.json がマージされます。

- override_1.json
- override_2.json
- override_3.json
- override.json 

-----------------------
lastWindowBounds.json
-----------------------
lastWindowBounds.json は自動的に作成されます。
ウィンドウの位置などが保存されています。
つまりこのファイルを削除することで初期位置に戻ります。


-----------------------
config_dump.json 
-----------------------
config_dump.json は、読み込まれません。
?target=server&cmd=dump_config によって出力されるものです。
現在の設定内容を確認したい場合に利用します。
参照専用です。
